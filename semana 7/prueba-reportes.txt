Al realizar una petición POST a la ruta /reportes, el servidor recibe la información enviada por el usuario en formato JSON y crea un nuevo reporte dentro de la aplicación. En este caso, se envió un reporte de tipo "Infraestructura" con la descripción "Daño en alumbrado público". Como respuesta, el servidor devolvió un objeto JSON indicando que el reporte fue registrado correctamente mediante el mensaje "Reporte registrado", junto con los datos almacenados y un identificador único (id: 2) asignado automáticamente al nuevo reporte.

Posteriormente, al realizar una petición GET a la misma ruta /reportes, el servidor devolvió un arreglo JSON con todos los reportes registrados hasta el momento. En la respuesta se observa el reporte inicial relacionado con un problema en la vía pública y el nuevo reporte de infraestructura agregado mediante la petición POST. Esto demuestra que la API permite tanto registrar nuevas incidencias como consultar el listado completo de reportes almacenados, facilitando la gestión y seguimiento de las situaciones reportadas por la comunidad.
